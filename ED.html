Ingeniería del software

La ingeniería del software es la disciplina que aplica principios, métodos y buenas prácticas de ingeniería para desarrollar software fiable, eficiente, mantenible y de calidad.

Nace como respuesta a la crisis del software (años 70), donde los proyectos fallaban por sobrecostes, retrasos y mala calidad.

Su objetivo principal es regular y ordenar el proceso de desarrollo para evitar errores y mejorar la calidad.

Tareas que incluye

  • Planificación del proyecto
  • Análisis y diseño del sistema
  • Programación
  • Pruebas, verificación y validación
  • Implantación y mantenimiento
  • Control de calidad, gestión de riesgos y documentación

Fases de desarrollo de una aplicación informática

Análisis

  • Se recopilan requisitos funcionales y no funcionales

Diseño

  • Se modela el sistema
  • Se define qué debe hacer el software
  • Se decide cómo se implementará la solución
  • Se definen componentes, arquitectura, base de datos, interfaces

Programación

  • Se escribe el código fuente siguiendo normas de estilo
  • Se generan ejecutables

Pruebas

  • Pruebas unitarias, de integración, de sistema y de validación
  • Se detectan y corrigen errores

Implementación

  • Instalación en el entorno del cliente
  • Pruebas beta y puesta en producción

Mantenimiento

  • Correctivo: corregir fallos
  • Adaptativo: adaptar a nuevos entornos
  • Perfectivo: añadir mejoras y nuevas funciones

Perfiles profesionales que intervienen en el desarrollo de software

En un proyecto intervienen varios roles especializados:

  • Jefe de proyecto: Responsable de la planificación, supervisión, control y coordinación del proyecto.
  • Expertos del dominio: Personas del área del cliente que conocen el proceso real. Ayudan a definir los requisitos.
  • Analista (analista funcional): Traduce las necesidades del cliente en modelos y requisitos claros.
  • Arquitecto de software: Define la arquitectura general del sistema y toma decisiones técnicas clave.
  • Diseñador: Detalla componentes, clases, interfaces y estructura interna del sistema.
  • Programador / desarrollador: Escribe el código fuente siguiendo las especificaciones del diseño.
  • Equipo de pruebas (QA): Realiza pruebas, reporta errores y asegura la calidad del producto.
  • Personal de soporte y mantenimiento: Ayuda al cliente y adapta el software con el tiempo.
